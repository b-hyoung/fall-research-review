## 1. 목적 & 배경

presentation.html Step 3의 추상적 연구 제안(mmWave+CSI 듀얼 센서)을 **개발 착수 가능한 수준의 구체적 플로우**로 전환한다. 지금의 flow-node 다이어그램(presentation.html:841–855)은 개념 수준이라 "실제로 CSI가 어떤 데이터를 뽑아 쓰고, mmWave가 어떻게 교차검증되는지"를 팀원·지도교수가 읽어낼 수 없다.

이 페이지는 **센서의 역할이 사용자 플로우에 자연스럽게 녹아드는가**를 최우선 기준으로 설계한다. 스토리와 기술 스펙이 분리되지 않고, 동일 시점에 환자/CSI/mmWave/운영자 4관점이 한 화면에 동시 표시되는 구조를 채택한다.

**핵심 방향**
- **메인 예측 축**: 보행 지표 (속도·보폭·cadence·STV)
- **보조 맥락**: 생체 신호 (호흡·심박·재실)
- **환경**: 1인 독실 (Yu 2025 독거노인 연구와 정합)
- **시나리오**: 하루 3시점 복합 (기상 baseline · 오후 오탐 회피 · 새벽 사전 예측)
- **표기 철학**: "**어떤 시스템으로 뽑는가**"가 1차 (파이프라인), 예시값은 2차(illustrative), 근거 논문은 3차. 값 자체가 아니라 추출 과정이 개발 착수의 핵심이므로.

**대상 독자**
1. 개발 착수 시 팀 내부 구현 합의용
2. 교수님 보고 시 설계 구체성 증빙

## 3. 페이지 아키텍처

### 전체 구조 (세로 스크롤)

```
1. 헤더
   제목: "사용자 플로우 — 독거노인 독실 · 하루 시나리오"
   부제: 환자 · CSI · mmWave · 운영자 4관점 동시 추적

2. 환경 브리핑 블록
   - 공간: 1인 독실 (요양시설 또는 자택)
   - 센서 배치: WiFi AP 1 + mmWave 레이더 1 + 간호스테이션 대시보드
   - 센서 역할 분담: CSI=연속 생체·재실, mmWave=정밀 동작·자세
   - 정확도 배지 범례 (✅ 📖 🎯)

3. 하루 타임라인 앵커 네비
   [ 🌅 07:15 기상 ] → [ ☕ 14:30 오후 ] → [ 🌙 03:42 새벽 ]

4. 장면 카드 × 3

5. 종합 블록
   - 3시점 데이터가 시계열 예측 모델로 합쳐지는 흐름
   - 평가 지표 (AUROC, F1, 선행 시간)

6. 참고문헌 섹션 (presentation.html 스타일 재활용)

7. 푸터: "← 발표모드로 돌아가기"
```

### 장면 카드 컴포넌트

각 시점은 **2×2 동시 표시 카드** 하나로 표현:

```
┌──────────────────────────────────────────────────────────┐
│  [아이콘] [시각] · [시점 이름]                            │
│  한 줄 요약                                               │
├──────────────────────────┬───────────────────────────────┤
│ 👤 환자 관점             │ 📡 CSI (WiFi 채널정보)        │
│ 서사체 스토리 2–3줄       │ [감지 항목 × N]               │
│                          │   ├ 시스템: 파이프라인        │
│                          │   ├ 예시값: X [illustrative]  │
│                          │   └ 근거: 논문 + 배지          │
├──────────────────────────┼───────────────────────────────┤
│ 📡 mmWave (FMCW 레이더)  │ 🖥️ 간호 스테이션 화면        │
│ [감지 항목 × N]          │ 알림 레벨                     │
│   ├ 시스템: 파이프라인   │ 표시 항목                     │
│   ├ 예시값: X            │ (교차검증 박스 조건부)        │
│   └ 근거: 논문 + 배지    │ 영상 없음 (RF만)              │
└──────────────────────────┴───────────────────────────────┘
                  ↓ (전환 메시지)
```

**원칙**
- 4분면 항상 동시 표시 (드로어·탭 없음 — 통합성 우선)
- 각 센서 칸 고정 구조 = **[감지 항목] → [추출 시스템] → [예시값] → [근거]**
- **시스템(파이프라인)을 1차 시각 초점**으로 배치: 예시값보다 위·굵게
- 예시값은 `[illustrative]` 라벨, 흐린 색상으로 보조 정보 처리
- 카드 간 화살표 + 전환 메시지로 시간 흐름 표현
- 새벽 시나리오 카드 우하단(운영자 영역)에 **교차검증 AND 박스** 추가 표시

**반응형**
- 데스크톱: 2×2 그리드
- 모바일: 1×4 세로 재배치
- CSS Grid로 구현 (JS 없음)

**시점별 색 포인트**
- 🌅 기상: 주황/앰버 계열 (정상 루틴)
- ☕ 오후: 그린 계열 (안정)
- 🌙 새벽: 블루/보라 계열 (경보)
- CSI=파랑 / mmWave=그린 (presentation.html 기존 색 유지)

## 4. 시나리오 콘텐츠

### 🌅 07:15 · 기상 (일일 baseline 갱신)

매일 반복되는 정상 루틴 — 생체·보행 지표 누적, 추세 업데이트.

**👤 환자**
수면 중이던 할머니가 천천히 몸을 일으켜 침대 가장자리에 앉음 → 기립 → 화장실 방향 이동.

**📡 CSI**

| 감지 항목 | 시스템 (파이프라인) | 예시값 | 근거 |
|---|---|---|---|
| 호흡 | Amplitude → Hampel 필터 → 서브캐리어 PCA → Band-pass 0.2–0.4Hz → FFT 피크 | 0.25Hz (15회/분) | 의학 표준 ✅ |
| 심박 | Phase → 고주파 Band-pass 0.8–2Hz → HRV 추정 (정지 자세 조건) | 1.1Hz (66bpm) | 의학 표준 ✅ |
| 재실/기상 | Amplitude variance 임계값 이벤트 | energy ↑ 검출 | 일반 CSI 문헌 📖 |
| 보행 cadence | Amplitude 시계열 주기 검출 → step count / time | 0.85Hz (51스텝/분) | Wang 2016 ✅ |

**📡 mmWave**

| 감지 항목 | 시스템 (파이프라인) | 예시값 | 근거 |
|---|---|---|---|
| 기립 시퀀스 | Point Cloud 수직 분포 클러스터링 → 자세 변화 추적 | 상체 분리 → 직립 | Yu 2025 📖 |
| 보행 속도 (메인) | Point Cloud → torso centroid 추적 → 시간 미분 + Doppler 평균 | 0.78 m/s | Zeng 2022 96% ✅ |
| 보폭 | Foot cluster 검출 → heel-strike 타이밍 | 0.48m | Zeng 2022 📖 |

**🖥️ 운영자**
알림 레벨 0. 일일 카드: "오늘 보행 속도 0.78m/s, 7일 평균 대비 −4%" (관찰 대상).

### ☕ 14:30 · 오후 (오탐 회피)

일상 활동(화장실·물 마시기) — 시스템이 "정상"으로 판정, 알림 없음.

**👤 환자**
물을 마시러 일어나 거실 이동, 화장실 다녀와 다시 앉음.

**📡 CSI**

| 감지 항목 | 시스템 (파이프라인) | 예시값 | 근거 |
|---|---|---|---|
| 호흡·심박 | 동일 파이프라인 상시 가동 | 0.28Hz / 1.2Hz(72bpm) | 의학 표준 ✅ |
| 보행 리듬 일관성 | Amplitude 주기 시계열 → 변동계수 계산 | 약 6% (낮음) | Wang 2016 기반 📖 |

**📡 mmWave**

| 감지 항목 | 시스템 (파이프라인) | 예시값 | 근거 |
|---|---|---|---|
| 자세 유지 | Point Cloud 수직 분포 → 직립 상태 지속 | 직립 100% | Yu 2025 📖 |
| 걸음 간격 | Heel-strike 타이밍 분산 | 일정 (정상 범위) | Zeng 2022 📖 |
| 낙상 signature | Point Cloud 급변 + Micro-Doppler pattern | 없음 | Yu 2025 ✅ |

**🖥️ 운영자**
활동 로그만 기록, 알림 없음. "정상 활동 패턴"으로 분류 → **False Positive 회피 증명**.

**이 카드 역할**: 단일 센서 연구의 오탐 문제 해결을 시연.

### 🌙 03:42 · 새벽 (사전 예측 트리거) — 핵심 시나리오

수일간 누적된 보행 악화 + 지금 이 순간 생체·자세 이상 → 낙상 발생 **전** 알림.

**👤 환자**
침대 가장자리에서 2분간 뒤척임 → 불안정 기립 시도 (졸음 + 방향 혼란).

**📡 CSI**

| 감지 항목 | 시스템 (파이프라인) | 예시값 | 근거 |
|---|---|---|---|
| 호흡 불규칙 | Band-pass 0.2–0.4Hz 출력의 시간창 분산·변동계수 계산 | 0.18–0.32Hz 변동 / CV 32% | 의학 표준 + 자체 임계 📖 |
| 심박 상승 | HRV 추정 + 평상시 baseline 대비 편차 | 1.5Hz (90bpm, ↑) | 의학 표준 ✅ |
| 미세 움직임 이상 | Amplitude energy 이상 패턴 지속 시간 검출 | 30초 지속 | 자체 임계 📖 |

**📡 mmWave (메인 예측 축)**

| 감지 항목 | 시스템 (파이프라인) | 예시값 | 근거 |
|---|---|---|---|
| 부분 기립 시도 | Point Cloud 상·하체 분리 클러스터링 → 움직임 불일치 감지 | 상체 움직임 / 하체 침상 | Yu 2025 구조 📖 |
| 미세 속도 | Doppler 저속 벡터 추출 | 0.08 m/s | Zeng 2022 기반 📖 |
| **STV 장기 추세** | 72시간 누적 cadence 시계열 → stride time 변동성 계산 | 변동성 상승 추세 | 유아현 2025 메타분석 ✅ |

**🔀 교차검증 AND 박스** (이 시나리오에서만 활성)

| 조건 | 값 |
|---|---|
| CSI 판정 | 생체·보행 리듬 "이상" 신뢰도 ≥ 0.8 |
| mmWave 판정 | 자세·속도·STV "이상" 신뢰도 ≥ 0.8 |
| 결과 | AND 충족 → **확정 알림** / 한쪽만 이상이면 "관찰 레벨"로 격하 |

**🖥️ 운영자**
**알림 레벨 2 — 낙상 위험**. 병실·시각·근거 표시 (영상 없이 수치·추세). "지난 3일 보행 속도 −8%, 지금 호흡 변동 ↑" 설명.

**위계**
- 1차 근거: 보행 변동성 STV 상승 추세 (메인 예측 축)
- 2차 근거: 생체 신호 불규칙 (보조 맥락)

## 5. 센서 스펙 정확도 정책

### 배지 3등급

| 배지 | 의미 | 사용 조건 |
|---|---|---|
| ✅ **검증됨** | 로컬 PDF 또는 공식 표준에서 직접 확인 | 원문 인용 가능해야 함 |
| 📖 **논문 범위** | 제시된 논문의 값 인용 (우리 환경 적용 여부 미지수) | 출처 링크 필수 |
| 🎯 **설계 목표** | 달성하고자 하는 목표치 (현재 측정치 아님) | "목표" 명시 |

**원칙**: 배지 없이 숫자 표기 금지. 환각 방지 최상위 정책 (presentation.html에서 Gowri 2025·복지부 "1조 6천억" 이슈 재발 방지).

**장면 카드의 구체 수치에 대한 규칙**: 시나리오 카드에 등장하는 개별 수치(예: 0.78m/s, 15회/분, 변동계수 32%)는 **설계 목적의 illustrative value**다. 각 값은 본 섹션의 verified 범위 내에서 선택되었으며, 페이지 상단 범례에 "장면 카드 수치는 예시이며, 괄호 안 범위·Hz 표기가 검증된 근거"라고 명시한다. 배지(✅/📖)는 해당 **범위/Hz 단위의 근거**에 부여되는 것이지, 개별 예시 값 자체의 실측 여부를 뜻하지 않는다.

  - 🌅 기상 = 주황/앰버
  - ☕ 오후 = 그린
  - 🌙 새벽 = 블루/보라
  - CSI = 파랑, mmWave = 그린 (기존 색 재사용)
- **배지**: ✅ 초록 / 📖 회색 / 🎯 오렌지 — 작은 알약형
- **화살표**: 카드 사이 세로 화살표, 전환 메시지를 위에 표기

**파일 규모 예상**: 600–900줄

## 8. 보류 항목 (구현 전까지 확정해야 함)

- **의학 표준 호흡·심박 범위 출처 논문**: 현재 "✅ 의학 표준"으로 표기된 범위(호흡 0.2–0.4Hz = 12–24회/분, 심박 0.8–2Hz = 48–120bpm)는 임상 가이드라인 출처 필요 (예: AHA, NEJM, Vital Signs 임상 기준 논문). 구현 시까지 확정 후 배지 옆에 인용 링크 추가.
- 배지는 일단 `✅ 의학 표준 [출처 확정 대기]` 형태로 placeholder 표기하고, 최종 단계(구현 9번 — 수치 교차검증)에서 일괄 반영.
- 기타 📖 배지가 붙은 항목 중 출처 애매한 것 (CSI 재실 감지 등) 동시 정리.
